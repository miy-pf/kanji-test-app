# かんじ練習カード

小学生が紙に漢字を書いて練習するための、全380問の暗記カード型HTMLアプリです。
手書き認識や自動採点は行わず、表示された答えと紙を見比べて自分で採点します。

## 使い方

1. 出題モードを選びます。
   - 学習する漢字は **新出漢字** または **文中のすべての漢字** から選べます
   - **全問**: 登録されている全問題
   - **10問特訓**: 各ページを1〜10番、11〜20番に分けて練習
   - **フラッシュカード**: 答えを確認しながら見るだけの練習。結果や履歴には反映されません
   - **全文**: 例文全体を読みで見て、全文を漢字かな交じりで書く練習
   - **テスト**: 現在の実力を測り、終了後に点数と間違いを確認
   - **A優先**: `priority: "A"` の問題
   - **段階別復習**: 復習が必要な問題を、超苦手・苦手・やり直しに分けて練習
2. 必要に応じて「ランダム」と「苦手な問題を先に出す」を選びます。
3. 「この設定ではじめる」を押します。
4. 画面の読みを見て、紙に漢字を書きます。
5. 「書けた」または「わからない」を選びます。どちらでも必ず答えが表示されます。
6. 「書けた」を選んだ場合は、漢字・送り仮名・字の形を見比べて
   「合っていた」または「間違っていた」を選びます。
7. 「わからない」を選んだ場合は、自動的に「×」として記録されます。

「新出漢字」は教材で指定された出題箇所を中心とする全380問です。
「文中のすべての漢字」は例文内の漢字語を一語ずつ別問題にします。
たとえば「昔の歌」は「むかしの歌」と「昔のうた」の2問として出題されます。
追加された漢字語も、それぞれ個別に学習履歴・習得・復習状態が保存されます。
この設定で「10問特訓」を選ぶと、10個の元例文に含まれる漢字語へ展開されるため、
実際の出題数は10問より多くなる場合があります。
ページや範囲の選択肢は `questions.js` のページ表記から自動で作られます。

「書けた」と思ったのに間違っていた問題は「思い違い」として記録され、
通常の「×」より優先して復習に出ます。
「×」は同じ練習の3〜5問後に一度再出題されます。
正答率と○・×の数には最初の回答だけを使い、再出題の回答は含めません。

段階別復習では、復習対象を次の3段階に分けます。
「やり直し」は×が1回だけの問題です。
「苦手」は×が2回、または「書けた」と思ったのに間違えた問題です。
「超苦手」は×が3回以上、または思い違いが2回以上の問題です。
復習モードでは「すべての復習」「超苦手だけ」「苦手以上」「やり直しだけ」を選べます。

同じ問題に別の日で2回「○」をつけると「習得済み」になります。
間違えた問題は、同じ日に再出題で正解しても復習対象に残り、
別の日に正解すると復習対象から外れます。

学習履歴はブラウザの `localStorage` に保存されます。
画面右上のユーザー切替で、`k`、`guest1`、`guest2`、`guest3` の4人を選べます。
学習履歴・累計履歴・テスト履歴はユーザーごとに別々に保存されます。
ログインや外部通信は行いません。
画面の「累計履歴」には、これまでの累計回答数・正答率・○・×・思い違い・
復習対象の問題数と、超苦手・苦手・やり直しの件数が表示されます。
ページの更新や新しい練習の開始では消えません。
「履歴リセット」を押すと、現在選んでいるユーザーの保存履歴だけを削除できます。
以前の単一ユーザー履歴がある場合は、初回のみ `k` に引き継がれます。

フラッシュカードモードでは、全380問または各10問区分を選び、
「答えを見る」「次のカードへ」の操作で確認できます。
○・×の採点は行わず、今回の結果・累計履歴・習得判定・復習対象は変更されません。

全文モードでは、全380文または各10問区分を選び、例文全体の読みを見て、
文全体を漢字かな交じりで紙に書きます。
たとえば「やまにのぼる」と出題され、「山に登る」と答えます。
送り仮名を含む語は、問題画面の読みの下線が波線で表示されます。
このモードでは「新出漢字 / 文中のすべての漢字」の設定は使わず、例文単位で出題します。
全文モードの履歴は通常の単語練習とは別に保存されます。

テストモードでは、全380問、A優先、各10問区分、ランダム10問・20問から
範囲を選べます。紙に書いて「回答した」を押し、答えを見て正解・不正解を選びます。
テスト中は途中成績を表示せず、その場で再出題もしません。
終了後に点数、正答率、間違えた問題一覧を表示し、間違いだけを通常学習で復習できます。
テストの正解は通常の累計正答率や習得判定へ加えません。
不正解の問題だけが通常の復習対象へ追加されます。
最近5回の結果は「テスト履歴」に表示され、最大50回分を保存します。

## 問題データの追加方法

`questions.js` の `QUESTIONS` 配列へ、次の形式で問題を追加します。

```js
{
  id: 11,
  page: "4",
  number: 11,
  question: "白一色の山",
  target: "白一色",
  answer: "白一色",
  reading: "しろいっしょく",
  priority: "A"
}
```

- `id`: 全問題で重複しない番号。履歴の保存に使います。
- `page`: 元教材のページ名。
- `number`: ページ内の問題番号。
- `question`: 元教材に書かれている問題文。
- `target`: 元の問題文で隠す部分。省略した場合は `answer` が使われます。
- `answer`: 紙に書く正しい答え。送り仮名を含めても構いません。
- `reading`: 問題画面に表示する読み。
- `priority`: `"A"` にするとA優先モードに出ます。その他は `"B"` などにできます。

問題数を増やしても、HTMLや `app.js` の変更は不要です。
既存問題の `id` を変えると、その問題の保存済み履歴を引き継げなくなるため注意してください。

「文中のすべての漢字」で使う個別の出題語は、同じファイル内の
`ALL_KANJI_TARGETS` に問題IDごとの配列として登録されています。
新しい例文を追加した場合、この配列にも文中の漢字語・答え・読みを追加してください。

## ローカルでの起動方法

フォルダ内の `index.html` をダブルクリックし、ブラウザで開きます。
サーバーやインストール作業は不要です。外部通信も行いません。

ブラウザの設定でローカルファイルの `localStorage` が無効な場合は、履歴を保存できません。
その場合は、簡易サーバーを使うかGitHub Pagesで公開してください。

## GitHub Pagesで公開する方法

1. GitHubで新しいリポジトリを作ります。
2. `index.html`、`style.css`、`app.js`、`questions.js`、`README.md` と
   `fonts` フォルダをアップロードします。
3. リポジトリの **Settings** を開きます。
4. 左側の **Pages** を開きます。
5. **Build and deployment** のSourceで **Deploy from a branch** を選びます。
6. Branchを `main`、フォルダを `/ (root)` にして **Save** を押します。
7. 表示された公開URLへアクセスします。

公開後も学習履歴は各端末・各ブラウザ内に保存され、ほかの利用者とは共有されません。

## フォント

答え表示には、アプリ内の `fonts/KleeOne-Regular.ttf` を使用します。
そのためSafari、Chrome、iPadなどでも、端末にインストールされたフォントに左右されず
同じ手書き風の字形で表示されます。フォントの読み込みに外部通信は使いません。
